What's Next?

When programming real hardware, the vast majority of programmers avoid writing bytecode. The sooner they can get at least an assembler up and running, the better. Many development systems even come from the manufacturer with C too. Bytecode is just impractical for a project of any significant size. Assemblers are not that hard to write, and make the programmer's life much less miserable.

Even the most basic of assemblers hold a critical advantage, labels. With bytecode, the code you write has to conform to fixed positions. If you want to return to a position in code you've been to before, you have to hardcode that memory location. Comments are also quite useful. With bytecode, there's only space for code and data. With assembly, you can select lines or parts of lines to ignore, and use that space to store information to clarify program specifics tothe reader.

Bytecode maintains one advantage. Self-modification. A clever programmer writing bytecode can take advantage of far more than the apparant usable space by storing new instructions on top of the old ones, and then re-executing the block. Unfortunately, while it's nice for space-saving, it's not great for the modern processor's caching systems. When you modify the code at runtime, the processor's cached bytecode becomes invalid, and it has to stop and pull in the new code. On top of that, it's not great for system security. If the program can modify itself, the user may be able to coerce the program to make malicious self-modifications, jumping into blocks of code that the program wasn't intended to run.

; INIT
start:
LOAD A 80 ; load x
LOAD B 81 ; load y
CMP A B ; is x > y?
JG swap ; goto swap

; 9 - SAVE_VAR
save_var:
STORE A 96 ; save min(x, y) as i
STORE B 112 ; save max(x, y) as increment
JMP loop

; 32 - SWAP
swap:
SWAP A B ; (x, y) -> (y, x)
JMP save_var

; 36 - LOOP
loop:
LOAD A 96 ; get i
ISZERO A ; is i == 0?
JZ exit
DEC A ; i--
STORE A 96 ; save new i
JMP add_increment

; 67 - ADD INCREMENT
add_increment:
LOAD A 97 ; read in cumulative value
LOAD B 112 ; read in increment
ADD A B ; cumulative = cumulative + increment
JERR exit ; if overflow, exit
STORE A 97 ; save new cumulative
JMP loop

; 255 - EXIT
exit:
HALT

; DATA
$80 6 4 ; x, y
